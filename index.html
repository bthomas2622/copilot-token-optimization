Best Practices

General strategies to reduce token consumption while maintaining output quality

Be Specific in Prompts

Vague prompts lead to multiple back-and-forth cycles. Each revision costs tokens. Front-load context by specifying language, framework, constraints, and expected output format upfront.

Prompt Engineering All Surfaces

Use Completions Over Chat

Inline tab completions and code comments that guide suggestions consume far fewer tokens than a chat round-trip. For small, well-defined code — a function, a loop, a type — write a descriptive comment and let Copilot complete it instead of opening chat.

Completions Low Token Cost

Scope Your Context Surgically

Never say "read the codebase" — that loads every file Copilot can see. Most tasks touch 2–3 files. Point directly to those with #file references. Close unrelated tabs. If you need to read a file yourself, read it yourself — don't ask the AI to load it into context on your behalf.

Context Window Precision

Break Down Large Tasks

Instead of asking for an entire feature in one prompt, decompose it into smaller, focused steps. Smaller requests produce better results with fewer retry tokens and reduce wasted output on wrong approaches.

Task Decomposition Quality

Accept & Edit, Don't Regenerate

If a suggestion is 80% right, accept it and manually edit the remainder rather than regenerating. Each regeneration re-reads the full context and produces a full new response — doubling the token cost for potentially marginal improvements. Small manual edits are free.

Workflow Token Savings

Understand Context Accumulation

Every message re-reads the entire conversation from the top. Message 40 costs far more than message 5 — not because it's harder, but because it re-processes everything before it. Tool outputs, file reads, and command results all stack permanently. New topic = new conversation. No exceptions.

Core Mechanic Session Hygiene

Use /compact Proactively

Don't wait until context is bloated. Run /compact after completing a sub-task — before starting the next one. It summarizes and compresses your history, keeping the thread alive but dropping the raw weight. This is the highest-leverage habit for long sessions.

Session Management CLI & Chat

Keep Session Notes

At the end of a session, ask Copilot to summarize key decisions and next steps into a short notes file. Start the next session by loading that file. You carry forward exactly what matters — not 200 messages of accumulated context. This lets you start fresh without losing progress.

Workflow Context Carry-Forward

Don't Use AI for Your Own Reads

If you need to read a log file, check an API response, or inspect data yourself — do it yourself. Every file the AI reads gets appended to context permanently for that session. Use your terminal, your editor, your browser. Keep the AI context clean for tasks that actually need AI.

Discipline Context Hygiene

Reduce Reasoning Effort

Extended thinking burns output tokens — sometimes tens of thousands per request. For tasks that don't need deep reasoning (renames, simple edits, quick lookups), lower the effort level. Don't under-invest on architecture decisions, but don't overspend on mechanical work either.

Thinking Tokens Output Cost

Write Caveman Prompts

The single most effective technique. Drop linguistic scaffolding that adds tokens without information: articles (a, an, the), filler (just, really, basically), pleasantries (Sure, I'd be happy to help!), hedging. Pattern: [thing] [action] [reason]. "Refactor function. Fix auth handling. Make efficient." conveys the same intent as a 40-token polite version — at 10 tokens. Real-world savings: 30–50% on input tokens.

Prompt Compression 30–50% Input Savings

Constrain Output Format

Output tokens cost 5× more than input tokens. One line in copilot-instructions.md — "Code only, no explanation" — cuts output cost 40–70% on every code task, permanently. Other high-value constraints: "Answer in one sentence", "3 bullet points max", "Yes or no, then one line why". Set terse output as a project default so you don't have to remember each time.

Output Control 5× Cost Multiplier

Compress Your Instructions File

Your .github/copilot-instructions.md is loaded on every Copilot interaction. A typical 200-line file burns ~1,500 tokens per call — before you type anything. Compress it the same way you'd compress a prompt: strip filler, use fragments, keep only landmines the agent can't discover on its own. A well-compressed 6-line file at ~50 tokens saves 72,500 tokens over 50 interactions vs. a bloated one.

Always-On Context High Leverage

Use Declarative Guardrails

Imperative instructions ("First read the file, then identify all public functions, then for each one check...") are long and fragile. Declarative guardrails ("All exported functions: JSDoc required.") are shorter, more stable, and compose cleanly. Five declarative invariants stack; five imperative procedures interfere. Use structured formats — bullets and key-value pairs beat paragraphs every time.

Tools

Token-efficient features and tools across each Copilot surface

VS Code — Inline Completions

Tab completions use far fewer tokens than chat. For small code snippets, let Copilot's inline suggestions do the work. Use Next Edit Suggestions and Copilot Edits for multi-file changes instead of chat-based iteration.

VS Code Completions

VS Code — Vision & References

Drag images into chat for UI work instead of describing layouts in text. Use #file, #selection, and #codebase references to provide precise context without pasting code blocks into the prompt.

VS Code Context Tools

VS Code — Agent Mode

Agent mode autonomously runs commands, reads files, and iterates — but every tool output stacks into context permanently. Each loop compounds the cost of every subsequent message. Set approval to Auto thoughtfully and watch for runaway loops. Reserve agent mode for tasks that genuinely need autonomy.

VS Code Agentic

Copilot CLI

Use sub-agents (explore, task) for parallel work — they run on smaller, cheaper models with isolated context. Use grep/glob/view tools directly for simple lookups. Know the difference: /compact summarizes and compresses your history; /clear wipes it entirely. Wrong choice = wasted tokens either way.

Terminal Sub-agents

Copilot Cloud Agent

Write detailed issue descriptions with clear acceptance criteria. The agent works autonomously on a VM — vague issues cause more exploration loops and token burn. Use copilot-setup-steps.yml to pre-install dependencies so the agent doesn't spend tokens figuring out the environment.

Cloud Autonomous

Model Selection

Default to Auto. Auto chooses from the supported model pool and applies a discount on paid plans — it's the best baseline. Override only when needed: pin a fast model (Claude Haiku) for renames and quick lookups, a standard model (GPT-4.1, Claude Sonnet) for daily implementation, or a premium model (Claude Opus) for architecture and deep reasoning. Teams switching from "always Sonnet" to "Auto, override when needed" generally reduce spend because they stop defaulting every interaction into the higher-cost lane.

All Surfaces Cost Control

Delegate to Sub-agents

When a task requires reading many files or running long analyses, delegate it to a sub-agent. Sub-agents run in separate sessions — their context accumulates independently. You get the result without their session weight landing in your main conversation. Anything touching more than 3–4 large files is a sub-agent candidate.

CLI Context Isolation

Exclude Files from Context

Use .copilotignore or equivalent ignore patterns to prevent Copilot from loading build artifacts, lock files, logs, and node_modules. Anything the AI doesn't need to see shouldn't be visible to it. But don't over-prune — excluding files Copilot actually needs causes worse suggestions and more retries.

Configuration One-time Setup

Prefer CLI Tools Over MCP

When a CLI equivalent exists, use it. A shell command returning 10 lines costs 10 tokens of context. The same request through an MCP server might return a structured JSON blob 100x larger. The full MCP output lands in context permanently. Over a session, the difference compounds dramatically.

MCP Token Reduction

VS Code — Copilot Edits

For multi-file refactors, use Copilot Edits instead of asking chat to generate code you then copy-paste. Edits applies changes directly across files in a single operation — fewer round trips, less context, and no manual transfer overhead. Add only the files the edit needs to touch.

VS Code Multi-file

Choose Ask, Edit, or Agent

Agent mode costs 5–25× more than Ask mode for the same prompt — it triggers 5–25 internal LLM calls with file reads, tool evaluations, and multi-step reasoning. "What does this function do?" → Ask (1 call, ~2K tokens). Single-file change → Edit (1–2 calls). Multi-file refactor with clear scope → Agent. Never use Agent for a vague prompt — clarify scope in Ask first, then switch.

Mode Selection 60–90% Savings

Scope Instructions with applyTo

Split one big copilot-instructions.md into a small always-on core plus scoped instruction files in .github/instructions/. Each file takes an applyTo frontmatter glob — Copilot only loads it when editing matching files. Your React questions stop paying for your database migration guidelines. Most teams have everything always-on; flipping the ratio cuts per-interaction context 50–80%.

Configuration Conditional Context

Audit Your MCP Servers

Each MCP tool definition costs 100–500 tokens — loaded every agent step. 10 servers × 5 tools × 200 tokens × 15 steps = 150K tokens just telling the agent what tools exist. Audit your enabled servers: if you don't need it for the current task, disable it. Use workspace-level .vscode/mcp.json for project-specific servers instead of enabling everything globally. VS Code's built-in tools already cover file read/write, terminal, and search.

AI Credit Tracking

Monitor your AI Credit usage at github.com/settings/copilot. As of June 2026, Copilot uses usage-based billing — every interaction consumes AI Credits based on token usage (input, output, and cached tokens) at published per-model rates. Each plan includes a monthly credit allowance (1 AI Credit = $0.01 USD). Code completions and Next Edit suggestions remain unlimited.

GitHub Settings Dashboard

Know Your Per-Token Costs

Different models have vastly different per-token rates. Output tokens cost 3–5× more than input tokens. Agent mode and Cloud Agent sessions consume far more tokens per task than chat or completions because they run multiple internal LLM calls. Factor this into your workflow — expensive models on high-value tasks, cheap models on quick questions.

Awareness AI Credits

Model Pricing Tiers

Every model has a different cost per million tokens. Lightweight models (GPT-5 mini, Gemini 3 Flash) are cheapest. Versatile models (GPT-4.1, Claude Sonnet, Haiku) are mid-range. Powerful models (Claude Opus, GPT-5.5, Gemini 2.5 Pro) cost the most. All usage draws from the same AI Credit pool — choosing the right tier for the task is the primary cost lever.

Build Personal Habits

Periodically check your AI Credit consumption and reflect on which workflows cost the most. Did you burn credits on long chat threads with expensive models? Could you have used inline completions instead? Small habit changes — like starting fresh conversations and choosing the right model tier — compound into big savings.

Set a Baseline First

Before optimizing, measure. Pick a representative task, note your AI Credits consumed, then apply optimizations and run the same task again. Without a baseline, you're guessing. The comparison is your signal — not generic percentage claims.

Measurement Before & After

Recognize Context Overload

Context overload doesn't always show up as a bill — sometimes it shows up as errors. Frequent internal errors, dropped connections, or "chat has reached its limit" messages are signs your context is too large. When you see these, don't retry — use /compact or start fresh.

Debugging Error Signals

Watch for Runaway Loops

Agent mode and sub-agents can enter loops — retrying failed commands, re-reading files, or exploring dead ends. Each iteration adds to context and costs tokens. If you see the AI repeating itself or spinning on an error, intervene manually. A 30-second human fix beats 10 rounds of AI retries.

Agentic Intervention

Prune Context Files Ruthlessly

Research shows LLM-generated context files (from /init or similar) hurt agent performance while increasing cost 20–23%. Agents discover project structure on their own — they already know how to ls and grep. Delete anything discoverable ("This is a Python project", "Tests are in tests/"). Keep only landmines: things that look normal but cause mistakes ("Use uv not pip", "Don't refactor the auth module"). Treat your context file like a bug tracker — it should grow and shrink, not just accumulate.

Context Files Research-Backed

Keep Commits & Reviews Terse

The Coding Agent reads git history and PR comments for context — every verbose commit or review comment costs tokens when ingested. Use Conventional Commits, subject ≤50 chars. For reviews, use one-line format with severity prefixes: "L42: 🔴 bug: user can be null. Add null guard before .email access." (~12 tokens vs ~40 for a paragraph). Savings seem small per-commit but compound across a repo's history.
