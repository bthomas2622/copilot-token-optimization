Ship More, Spend Less

A developer's guide to maximizing token efficiency across every GitHub Copilot surface

Foundation

Best Practices

General strategies to reduce token consumption while maintaining output quality

Be Specific in Prompts

Vague prompts lead to multiple back-and-forth cycles. Each revision costs tokens. Front-load context by specifying language, framework, constraints, and expected output format upfront.

Prompt Engineering All Surfaces

Use Custom Instructions

Set persistent context via .github/copilot-instructions.md so Copilot understands your project conventions without you repeating them. This eliminates redundant context in every prompt.

Configuration Context Reduction

Scope Your Context

Only include relevant files and code snippets. More context ≠ better results — it means more input tokens. Close unrelated tabs, use #file references to pinpoint exactly what Copilot should look at.

Context Window Efficiency

Break Down Large Tasks

Instead of asking for an entire feature in one prompt, decompose it into smaller, focused steps. Smaller requests produce better results with fewer retry tokens and reduce wasted output on wrong approaches.

Task Decomposition Quality

Accept & Edit, Don't Regenerate

If a suggestion is 80% right, accept it and manually edit the remainder rather than regenerating from scratch. Each regeneration consumes a full round of tokens for potentially marginal improvements.

Workflow Token Savings

Start New Conversations

Long chat threads accumulate context that gets re-sent with every message. When switching topics, start a fresh conversation. This prevents bloated context windows and keeps token usage lean.

Surface-Specific

Tools

Token-efficient features and tools across each Copilot surface

VS Code — Inline Completions

Tab completions use far fewer tokens than chat. For small code snippets, let Copilot's inline suggestions do the work. Use Next Edit Suggestions and Copilot Edits for multi-file changes instead of chat-based iteration.

VS Code Completions

VS Code — Vision & References

Drag images into chat for UI work instead of describing layouts in text. Use #file, #selection, and #codebase references to provide precise context without pasting code blocks into the prompt.

VS Code Context Tools

VS Code — Agent Mode

Agent mode can autonomously run terminal commands, fix errors, and iterate — but each loop costs tokens. Set Auto approval thoughtfully. Use @workspace for codebase-wide questions so Copilot indexes efficiently rather than scanning everything.

VS Code Agentic

Copilot CLI

Use sub-agents (explore, task) for parallel work — they use smaller models and less tokens. Leverage grep/glob/view tools directly for simple lookups instead of launching agents. Use /compact to summarize and reduce conversation context.

Terminal Sub-agents

Copilot Cloud Agent

Write detailed issue descriptions with clear acceptance criteria. The agent works autonomously on a VM — vague issues cause more exploration loops and token burn. Use copilot-setup-steps.yml to pre-install dependencies so the agent doesn't spend tokens figuring out the environment.

Cloud Autonomous

Model Selection

Choose the right model for the job. Use fast/cheap models (GPT-4.1, Claude Haiku) for simple tasks like code completion and quick questions. Reserve premium models (Claude Opus, o3-pro) for complex reasoning where quality matters most.

Observability

Monitor Your Usage

Track and understand your Copilot consumption to stay in control

Premium Request Tracking

Monitor your premium request usage at github.com/settings/copilot. This dashboard shows your monthly allowance and current consumption. Premium requests are used when you select premium models or use agentic features.

GitHub Settings Dashboard

Organization-Level Insights

Org admins can view usage across the organization via the Copilot usage dashboard and the Copilot Metrics API. Track adoption, active users, and feature-level engagement to identify where tokens are being consumed across your team.

Admin API

Premium vs. Standard Requests

Understand what triggers a premium request: using premium models (Claude Opus, o3-pro, Gemini 2.5 Pro), agent mode sessions, and Copilot Cloud Agent runs. Standard models like GPT-4.1 and Claude Sonnet are included in your base allowance and don't count as premium.

Billing Models

Set Usage Policies

Organization admins can configure policies to control which Copilot features are available. Disable features your team doesn't need to prevent unintentional token spend. Manage editor-based and platform-based suggestions independently.

Governance Policies

CLI Session Awareness

In Copilot CLI, use /cost to see token usage for the current session. Use /compact regularly to compress conversation history and free up context window space. Watch for long-running agent loops that may burn through tokens.

CLI Real-time

Billing & Plan Management

Review your plan's included premium requests at github.com/settings/billing. Copilot Free includes limited premium requests, Pro includes more, and Enterprise/Business plans offer higher limits. Consider upgrading if you consistently hit limits.
